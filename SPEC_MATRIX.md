# Yei 1차 확장 규격 매트릭스 — 추천 10개국

> 한국 + 미국 · 셰겐 · 영국 · 일본 · 중국 · 인도 · 캐나다 · 호주
> 확인 기준일: **2026-06** (개별 페이지 인덱싱일은 출처표 참고)
> ⚠️ 이 문서는 **구현 전 레퍼런스**. 코드에 박기 전 각 항목을 해당국 정부 포털에서 재확인.
> 현재 보유: 한국(여권/면허/일반) · **미국 2×2** · **셰겐 35×45** (`src/usage.ts`).

---

## ⚠️ 정확도 주의 (출시 전 반드시 읽기)

1. **인쇄 크기·머리 비율·배경색·안경 규정**은 정부 페이지 또는 정부 기준을 미러링한 신뢰도 높은 출처로 교차 확인됨 → 신뢰도 **높음**.
2. **디지털 업로드 px·파일 용량**은 일부가 전문 사진 서비스(2차 출처)에서 가져온 값 → **각국 정부 업로드 포털에서 직접 재확인 후 코드에 박기**. 특히 자동검증 포털(중국 COVA, 인도 Passport Seva, 영국 GOV.UK 체커)은 밴드를 1KB·1px 단위로 거부함.
3. **안경 규정이 최근 빠르게 강화**됨 — 한국(2024 전면금지), 인도(2025.9 금지), 캐나다(2017 금지). 이 카테고리는 분기마다 재확인 권장.
4. Yei는 "**보장/100% 통과**" 워딩 금지. 이 표는 **정확성 포지셔닝**의 근거 데이터일 뿐 합격 보증이 아님.

## Yei 제품 관점 — 즉시 반영할 플래그

- **캐나다 여권은 DIY 앱으로 커버 불가.** 상업 사진가 촬영 + 뒷면 스튜디오 스탬프 + 보증인 서명이 의무라 온라인 갱신조차 사진관 디지털 파일을 요구. → Yei는 **캐나다 비자(35×45)만** 안전하게 지원. 여권은 "지원 안 함" 또는 "사진관 안내"로 처리.
- **영국은 흰색 배경이 오답.** 밝은 회색/크림이 정답 → 배경 프리셋이 국가별로 흰색 고정이면 영국에서 반려.
- **중국은 33×48 + 순백색 + JPEG only + 자동검증.** 셰겐 35×45와 절대 호환 안 됨. 별도 프리셋 필수.
- **인도는 한 나라가 3종**(여권 35×45 / OCI 51×51 / PAN 25×35) + 2025.9 안경 금지·ICAO 전환. 신청처(국내 PSK vs 미국 CKGS)별 인쇄/디지털 분기까지 있음.
- **안경 토글이 나라별로 달라야 함.** 한국·영국·인도·캐나다·호주·미국 = 금지(또는 사실상 금지), 일본·중국·셰겐 = 조건부 허용. 단일 "안경 OFF"로 묶으면 일본 등에서 과잉 제거.
- **편집/합성 게이팅**(표 3)이 passport-mode 외에 **국가 단위로도** 걸려야 함. 미국 프리셋 = 모든 합성 기능 강제 OFF.

## 묶음(템플릿) 재확인 결과

- **35×45 흰배경 묶음**으로 안전: 한국 · 셰겐(흰색 선택 시) · 일본 · 호주. → 머리 비율·안경만 파라미터로 분기.
- **묶음에서 빠져야 하는 예외:** 영국(회색/크림) · 미국(2×2) · 중국(33×48 순백) · 캐나다 여권(50×70) · 인도 여권(머리 36–38mm로 비정상적으로 큼).

## 🔧 구현 매핑 메모 (`src/usage.ts` `UsageSpec` 기준)

**기존 필드로 커버되는 것:** `widthMm/heightMm/dpi/targetW/targetH`(인쇄 크기), `faceMin/faceMax`(머리 비율 %), `eyeMin/eyeMax`(눈높이 밴드), `maxKB`(디지털 용량캡), `restricted`(편집 OFF), `premium`(해외 잠금), `notice`(안내문).

**구현 시 추가로 필요할 필드 (현재 없음):**
- `bgColor` / `bgGuide` — 배경색이 흰색이 아닌 경우(영국 회색/크림, 일본 파랑 허용, 중국 순백 강제). 현재는 흰색 가정 + notice 텍스트뿐.
- `glasses: 'forbidden' | 'conditional'` — 안경 규정 국가별 분기(표1) → notice/가이드 문구·경고에 사용. 단일 토글로 묶지 말 것.
- **디지털 ≠ 인쇄 px 분리** — 중국(인쇄 33×48mm지만 디지털 354×472~420×560px), 인도(630×810) 등은 인쇄용 px와 디지털 업로드 px가 다름. 현재 `targetW/H` 하나뿐 → `digital: {w,h,maxKB,format}` 서브스펙 또는 별도 출력 모드 필요.
- `format` — 대부분 JPEG지만 중국·인도는 **JPEG only**(PNG 거부) 명시 필요.
- 신청처/문서 다중(인도 3종, 미국 여권 vs 비자, 캐나다 여권 미지원) — `Usage` 키를 `in_passport`/`in_oci` 식으로 분리하거나 그룹 처리.

**안전장치:** 자동검증 포털(중국 COVA·인도 PSK·영국 GOV.UK)은 1KB·1px 단위로 거부 → 해당 규격은 `encodeToTargetSize`로 밴드 **상한 살짝 아래** 타겟(예: 120KB면 110KB) + px 정확. 출시 전 실제 포털에 1회 업로드 검증 권장.
