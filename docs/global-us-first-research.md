> 작성: 2026-06-25 · 결정된 범위: **영어권(미국) 우선** (영국·EU는 후순위 — 규격·법무 난이도 큼).
> 성격: **조사·결정 문서.** 코드 구현 아님. 각 항목은 "조사해서 값/판단을 확정"하는 게 목표.
> 근거 문서: `BACKLOG.md`(4-A 규격 티어 / 4-B 기술·컴플라이언스) · `SPEC_MATRIX.md`(미국 행) · `MAINTENANCE.md`(규격 점검).
> 원칙(유지): "보장/100% 통과/공인" 워딩 금지 · 사진 온디바이스·비수집 · 명의=개인.

## 왜 미국 우선인가 (전제)
- 미국 여권 **2×2in 프리셋은 이미 보유**(`src/usage.ts`) → 규격 신규 개발 최소.
- **명의=개인 리스크 최소**: EU DSA 같은 *개발자 주소 공개* 의무가 미국엔 없음.
- 진짜 신규 작업은 **i18n(영어 UI) + 영어 스토어/법무 표현**에 집중됨.

## 1. 시장·경쟁 (왜 이 시장에 돈이 되나)
- [ ] 미국 증명사진 수요: 여권 신규/갱신 규모, **State Dept 온라인 업로드 비중** vs 약국/USPS 인쇄 의존도.
- [ ] 경쟁앱 조사(미국 Play 상위 "passport photo"): 가격 모델(단건·구독·인쇄배송), 평점, **약점**(거부 후기 등).
- [ ] 차별점 정리: **1회 결제·구독 없음·온디바이스(사진 전송 없음)** = Yei 강점 → 카피로.
- [ ] 결과물: "미국 진입 근거 1쪽" (수요 + 경쟁 빈틈 + 가격대).

## 2. 규격 데이터 (보유분 재확인 위주 — 해자)
- [ ] **미국 여권 2×2in**: 흰배경·안경 금지(2016~)·머리 25–35mm → `SPEC_MATRIX.md` 값을 **State Dept 공식 포털에서 재확인**.
- [ ] **디지털 업로드 규격 재확인(중요)**: State Dept 온라인 제출 px·용량·포맷(자동검증이라 1px·1KB도 거부) → `usage.ts` `maxKB`/px 점검.
- [ ] (결정) **미국 비자 DS-160(2×2, 머리 50–69%)** 프리셋 추가할지 — 여권과 머리비율 다름.
- [ ] 확정값은 `MAINTENANCE.md` 점검표에 "미국" 행으로 편입(최종확인일 기록).

## 3. i18n — 영어 UI (최대 코드 작업, 별도 본인 확인 후 착수)
- [ ] 현재 **문자열 전부 한국어 하드코딩**(i18n 라이브러리 0) → 외부화 범위 산정: `App.tsx`·`Paywall.tsx`·패널들·`usage.ts`(label/notice)·`legal.ts`.
- [ ] 언어 감지(기기 locale) + 폴백 정책: 자동 전환 vs 한/영 토글.
- [ ] **법무 문서 영어판**: 개인정보처리방침·약관·FAQ — 단순 번역 아니라 영어권 표현(`legal.ts` + 호스팅 `privacy.html` 영어 URL).
- [ ] 단위 표기: mm ↔ **inch**(미국 익숙) 병기 검토.
- [ ] (주의) 아랍어 등 RTL은 1차 **제외**.

## 4. 법무·컴플라이언스 (미국 — 부담 작음)
- [ ] **CCPA(캘리포니아)**: 사진 온디바이스·비수집·판매 없음 → 부담 낮음. 단 개인정보처리방침에 캘리포니아 권리·연락 경로 명시.
- [ ] **COPPA**: 13세 미만 타깃 아님 명시(연령등급).
- [ ] Play **Data safety**: 기존과 동일(사진 수집 안 함) — 영어 표현만 점검.
- [ ] 명의/주소: 미국은 주소 공개 의무 없음 → 개인 명의 유지 OK. 개발자 연락 이메일·정책 URL은 보유분 사용.
- [ ] **"정부 공인/합격 보장 아님" 영어 고지** 유지.

## 5. 결제·가격·세금 (미국)
- [ ] **USD 가격 확정**: 경쟁앱 가격 조사 후(예: $3.99 / $4.99 …). 상품/entitlement는 동일(`yei_lifetime`/`premium`), 가격만 Play 지역별 현지화.
