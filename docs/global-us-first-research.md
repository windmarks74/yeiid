## 왜 미국 우선인가 (전제)
- 미국 여권 **2×2in 프리셋은 이미 보유**(`src/usage.ts`) → 규격 신규 개발 최소.

- 진짜 신규 작업은 **i18n(영어 UI) + 영어 스토어/법무 표현**에 집중됨.

## 조사 결과 (2026-06-26) — §1·§2 완료 · 코드 0

> 두 갈래 웹 리서치(시장·경쟁 / 규격 검증, 공식·1차 출처 우선) 결과. **판정: 조건부 GO.**

### 결론 한눈에
- **시장(§1):** 매력 분명 — 미국 여권 연 **2,450만~2,730만장 발급**, **온라인 갱신 2025년 정식 출범(1년차 200만+)으로 "집에서 찍어 업로드"가 제도화.** 단 무료 앱이 시장을 채워 **포화**.
- **규격(§2):** 우리 미국 프리셋은 인쇄·배경·안경·AI금지 전부 **공식 일치(정확)**. 단 단일 `us` 프리셋이 여권+비자를 섞어 **2곳(머리 mm 주석·maxKB 240)** 부정확 → `us_passport`/`us_visa` 분리 권장.
- **⚠️ 중대 발견:** 미국 여권은 **AI/필터 편집 금지(2025.10 강화)·모든 사진 AI 검출**. 우리 앱은 이미 `restricted` 모드로 미국에서 배경교체·뽀샤시를 끄므로 **준수 상태**. 단 이는 미국 여권의 가치제안이 "AI 배경 마술"이 아니라 **정확한 크기·포맷(온라인 업로드용 ≤240KB/600px)·인화시트**임을 뜻함 → 마케팅 메시지를 거기에 맞춰야 함.
- **가격:** **$4.99 1회결제(평생)** 권장(보수적이면 $3.99 진입). 한국 ₩4,900(≈$3.6)과 정합.
- **Go/No-Go:** **저비용·유기(ASO) 진입 = GO / 대규모 유료광고 선투자 = No.** $4.99 영어 MVP 소프트런치 → "구독 없음·온디바이스·규격정확" 카피로 유기 전환율·반려율 데이터 본 뒤 확전.

### §2 규격 — 공식 확정값 (코드 반영 시 근거)
- **미국 여권(인쇄+온라인 갱신):** 51×51mm · 머리 **25–35mm** · 흰/오프화이트 · 안경 금지(의료예외) · 6개월 이내 · **AI/필터 편집 금지**. 온라인 디지털 = **px 무규정**, 포맷 JPG/PNG/HEIC/HEIF, **54KB–10MB**(현 `maxKB:240`은 불필요하게 빡빡).
- **미국 비자(DS-160):** 51×51mm · 머리 **22–35mm(50–69%)** · **600×600~1200×1200 정사각 · JPEG 단독 · ≤240KB · sRGB 24bit · 압축 ≤20:1.**
- 출처(공식): travel.state.gov — `passports/how-apply/photos.html` · `.../online-renewal-photo.html` · `us-visas/.../photos.html` · `.../photos/digital-image-requirements.html`.

### usage.ts 수정 후보 (코드변경 → v13, 본인 확인 후)
- `us` 프리셋 주석 "머리 25~35mm" = 여권 기준임을 명확히(faceMin 50%는 비자 22mm 유래라 혼동).
- `maxKB:240`은 비자값 → 여권 단독엔 완화/제거, 포맷에 PNG/HEIC 허용 검토.
- **`us_passport` / `us_visa` 분리**(자동검증 반려 방지).

## 조사 결과 2 (2026-06-26) — §3~§6 완료 · 코드 0

### §3 i18n 영어화 분량 (코드 점검)
- **UI 문자열 ~130–170개**(App.tsx에 절반↑ 집중, 동적 템플릿 다수) + **법무 3문서 ~6–7천 자(별도 트랙)**.
- **권장: 경량 자작** — `strings.ts`에 `{ko,en}` 키맵 + 보간 `t()` + 로케일 감지(`navigator.language`). react-i18next는 과함.
- **법무는 분리 트랙:** 개인정보·약관·FAQ 영문 + **공개 `privacy.html` 영문본 호스팅** 필요(미국 맥락 재작성 — 한국 개인정보보호법 인용 제거).
- ⚠️ **결제코드 충돌 주의:** `₩4,900`이 `iap.ts`·`App.tsx`·`Paywall` 5+곳 하드코딩 → 영어판은 스토어 현지화 가격(`product.priceString`) 사용 필요. **결제·스토어 영역이라 본인 확인 필수.**
- 기타 리스크: mm→inch 환산 표시, 영어가 길어 칩/버튼 레이아웃 깨짐, 시험 프리셋(큐넷 등) 영어판 미노출 분기.

### §4 법무·컴플라이언스 (※ 법률자문 아님 — 공식/전문가 확인 권장)
- **CCPA/COPPA: 구조적으로 적용·부담 최소**(온디바이스·비수집·비아동). 영어 방침에 짧은 "California Privacy" 단락 + 타깃연령 13세+ 설정 + Data Safety 한국 신고 영어 복제로 충분.
- **⚠️ 개인 명의 주소 공개(가장 중요):** Google Play는 개인 계정이 **유료앱이면 개발자 실명 + 전체 주소를 스토어에 공개 표시**. EU만 아니라 **미국·글로벌에도 적용**(이미 한국 유료 출시라 현재도 해당 가능 — 현 리스팅 확인 권장). 부담되면 **개인사업자(법인 아님) 주소 사용** 검토 — 본인·세무 판단.

### §5 결제·세금
- **미국 판매세 = Google이 자동 처리**(개발자 직접 납부 불필요).
- **W-8BEN 제출 필요**(한국 개인) + 한·미 조약 청구. 원천징수율 = royalty 15%(문학·예술 10%) / **앱매출이 사업소득이면 0%일 수도** → **세무사 확인 필수(미확인).**
- **$4.99 기준가 1회 입력 → Play 자동 현지화.** RevenueCat entitlement·상품 그대로(미국용 별도 상품 불필요).

### §6 운영·스토어/ASO
- **영어 리스팅 추가**(제목·설명·**영어 스크린샷**) + **영어 개인정보 URL** 필수.
- ASO 키워드: passport photo / 2x2 photo / visa photo / passport photo maker / **on-device·private·no upload**(프라이버시 차별화).
- 출시국가에 US 추가, $4.99 확인. EU 제외하면 DSA trader 공개는 회피(미국 신원공개는 별개).
