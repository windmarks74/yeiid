# Yei 출시 후 마케팅·세일즈 플레이북 (초안)

> 출시 직후~초기 성장 단계 실행 가이드. 유료 광고보다 **유기적(ASO·SEO·평점)** 중심.
> 가격·키워드는 시작점이며, 데이터를 보고 조정.

## 1. 가격 설계 (티어)
**v1 (한국, 지금)**
- 무료: 국내 규격 5회 저장 (인화 시트·해외 규격 잠금)
- 프리미엄 1회 ₩2,900: 무제한 저장 + 인화 시트

**v1.1 (해외/미국 규격 추가 후) — 2티어 권장**
- 기본 ₩2,900: 국내 무제한 + 인화 시트
- 프리미엄 ₩6,900(~$5): + 해외·비자 규격 (지불의사 높은 비자 수요 겨냥)
- (단순하게 갈 거면: 단일 프리미엄을 ₩4,900으로 올려 해외 규격 포함)

**전환 레버**: "한 번에 통과 · 불합격 시 환불" 보장(leqi 검증). 비자 티어에 특히 효과. (환불은 스토어 정책/수동으로 운영.)
**지역 가격**: 플레이 가격 템플릿으로 시장별 현지가(예: 미국 비자 $4.99).

## 1-A. 1회 결제 전략 심화 (2026-06)

> 현재 라이브 모델 = **₩4,900 단일 1회 결제(`yei_lifetime`, entitlement `premium`) + 무료 5회 저장.**
> (※ 위 §1의 ₩2,900/2티어 표기는 초안값 — 실제 출시값과 다르니 차후 §1 본문도 ₩4,900 기준으로 정리 필요.)

### 평가 — 이 업종엔 1회 결제가 정답
증명사진은 **저빈도·고의도** 상품(여권 10년에 1번, 시험·면허 가끔). 이 특성이 전략을 규정한다.
- **구독은 독.** 1년에 한두 번 쓸 걸 월정액으로 묶으면 전환이 죽는다. "한 번 결제, 평생"은 오히려 **광고 카피 자산**(한국=구독 피로 큼).
- **₩4,900은 마찰 낮음.** 사진관·즉석사진(₩1만~2만) 대비 "더 싸다"가 즉시 성립 → 충동구매 구간.
- 결론: **구독으로 바꾸지 않는다.** 방향 유지.

### 레버 ① (최대) 무료 구간을 "물리게" 보정
1회 결제 모델의 진짜 약점은 가격이 아니라 **무료가 너무 후하면 단일 니즈 사용자가 페이월을 영영 못 보고 공짜로 떠나는 것.** 여권사진 1장은 최종 저장 1~2번이면 끝인데 무료 5회면 다수가 페이월 미도달.
- **검증 지표: "페이월 노출률"**(결제 후보 중 페이월을 보는 비율). 10~15% 미만이면 무료가 헐겁다는 신호 → 조인다.
- **더 강한 구조:** "완성 미리보기는 무료 → **내보낼(저장) 때 결제**" 또는 **무료=워터마크.** 결과물을 눈으로 본 직후가 지불의향 최고조 → 이 업종 최고 전환 구조.
- **균형:** 신규 앱은 평점·입소문이 생명 → **출시 직후엔 5회 유지, 데이터 보고 조이기.**

### 레버 ② LTV 천장은 구독 말고 "1회 결제 팩"으로 푼다
1인당 상한 ₩4,900을 푸는 길 = 가격 사다리(이미 코드에 `premium` 게이팅·시험 프리셋 로드맵 존재).
- `기본 ₩4,900(국내)` + `해외/비자 팩` + `시험 팩` 을 **추가 1회 결제 SKU**로 → 한 사용자가 여러 번 1회결제 → LTV↑(구독 0).
- (선택·테스트) 바닥에 **단건 ₩1,900 "이 사진 1장"** 소비형 → ₩4,900 안 살 1회성 고객 포획 + 평생권을 "더 이득"으로 앵커링. 단 복잡도·잠식 주의.

### 레버 ③ 가격은 가치 대비 저평가 가능
₩4,900은 "앱 기능값"이 아니라 **"사진관 ₩1만5천 trip 대체값"** → 싼 편. traction·전환 데이터 쌓이면 **₩6,900~9,900 A/B** 시도 가치. (출시 초기 traction 전엔 동결.)

### 한계 / 주의
- **프라이버시 우선 ↔ 최적화 데이터의 긴장.** 사진 무수집·온디바이스는 데이터안전 자산이지만 퍼널 데이터가 얇다. RevenueCat이 전환율·ARPU는 주니, **최소 "페이월 노출률"·"무료 소진 분포"는 집계로라도 보이게** 해야 레버①을 감 아닌 숫자로 결정.
- **환불보장 레버**는 전환엔 강력하나 1인 운영엔 어뷰징·수작업 리스크 → 노출·조건 좁게.

### 한 줄 요약
**1회 결제 유지 + "무료를 조여 페이월 물리기 · 완성순간 결제 · 1회결제 팩으로 LTV 확장"** 이 3개가 최대 레버. 가격·게이팅 변경은 심사/매출 직결 → 실데이터 + 본인 확인 후 반영.

## 1-B. 용도별 가치 메시지 (포지셔닝) — 2026-06

규제 차이를 그대로 "가치 메시지"로 전환 (앱 로직과 일치: 여권=`restricted` 보정OFF, 면허·일반=자유).
- **여권/비자(restricted):** "원본 그대로 **안심**" — 배경교체·강한보정 OFF로 반려 막기. → **신뢰** 메시지.
- **면허증·일반(자유):** "잡티 살짝 완화 + 깔끔한 배경 → 인상이 **산뜻·젊어 보이게**." → **wow/감정** 메시지 = 지불의향·입소문 동력. (이 앱의 진짜 차별점: 남들은 "규격 통과"만 말함.)
- **정직 가드레일:** 면허도 신분증 → 갱신 시 본인 대조 → **"자연스럽게, 본인 확인되는 선"**. "10년 젊게" 류 과장 금지(역효과·신뢰 손상).
- **적용처:** 스크린샷 **2번째 컷**(면허 before/after, "잡티만 완화해도 산뜻"), ASO 부제, SEO 글("면허사진 잘 나오게 찍는 법").

## 2. ASO (앱스토어 최적화) — 가장 중요한 무료 채널
- **제목/부제**에 핵심 키워드: 예) "Yei – 여권·증명사진·비자사진"
- **키워드(한국)**: 여권사진, 증명사진, 반명함, 비자 사진, 미국 비자 사진, 여권사진 규격, 면허증 사진, 증명사진 만들기
- **키워드(미국, 확장 시)**: passport photo, passport photo maker, visa photo, 2x2 photo, ID photo, DV lottery photo
- **키워드(일본)**: 証明写真, パスポート写真, ビザ写真, 履歴書写真
- **스크린샷**: 앞 2장이 승부. 가치 강조 — 규격 정확 / 흰 배경 / 인화 시트 / 합격 안심, 텍스트 오버레이.
- **현지화**: 진출 시장별 리스팅 번역.

## 3. SEO 콘텐츠 (검색 유입 깔때기)
- 고의도 검색을 잡는 가이드 글 → 앱으로 유입. 예: "미국 비자 사진 규격·만드는 법", "여권사진 규격 총정리", 나라별.
- 간단한 사이트/블로그에 호스팅(FAQ·개인정보처리방침도 같은 곳에).
- 검색해서 들어온 사람은 의도가 높아 전환율이 좋음.

## 4. 평점·리뷰 엔진
- 앱 내 평점 요청을 **만족 순간(저장/인화 성공 직후)**에, 과하지 않게(네이티브 in-app review).
- 리뷰에 답글(초기 수동) → 평점↑ → 랭킹↑ → 설치↑.
- 불만/버그는 빠르게 업데이트로 대응(업데이트 무료).

## 5. 타이밍·시즌
- 수요 급증기에 맞춰 노출·피처링: 유학(F-1) 시즌, 여행 성수기, 여권 갱신 주기.

## 6. 채널 우선순위
1. ASO (무료, 최우선)
2. SEO 가이드 콘텐츠
3. 리뷰·평점
4. 커뮤니티(유학·비자·여행 카페 등) — 도움되는 참여, 스팸 금지
5. (나중) B2B — 유학원·비자 대행사·이민 변호사 묶음/리퍼럴

> 유료 광고는 후순위: 싼 1회 결제라 보통 본전(LTV<CAC)이 안 나옴. 검증된 LTV가 생기기 전엔 지양.

## 7. 측정 (Play Console + RevenueCat)
- 설치, 스토어 전환(방문→설치), **설치→유료 전환**, ARPU, 평점 — 키워드·시장·국내/해외 규격별로.
- 전환율이 낮으면 게이트/가격/스크린샷 A/B.
