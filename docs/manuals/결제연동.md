# 매뉴얼 — 결제 연동 (RevenueCat + 구글 플레이 인앱결제)

> 목적: 다음에 또 결제를 붙일 때 **그대로 따라 할 순서** + 이번에 **어디서 막혔고 어떻게 풀었는지**.
> 앱: Yei (`com.yei.idphoto`) · 상품 `yei_lifetime`(일회성 ₩4,900) · entitlement `premium`.
> ⚠️ 이 문서는 코드·`IAP_SETUP.md` 에서 추출한 초안. **〔인터뷰〕** 표시는 본인 경험으로 채울 칸.

## 0. 핵심 제약 (먼저 이해)
- 구글 결제는 **(1) 앱이 플레이 트랙(내부 테스트)에 올라가 상품이 존재**하고
  **(2) 폰 설치본의 서명키가 업로드본과 일치**해야 동작.
- **USB 디버그 설치(sideload)로는 결제가 안 됨** → 반드시 **내부 테스트 링크로 설치**.
- 코드(`src/iap.ts`, `Paywall.tsx`, `billing.ts`)는 이미 연동됨. 막히는 곳은 늘 **콘솔/키 설정**.

## 1. 그대로 따라 할 순서

### A. 서명된 AAB 만들기
```
npm run sync:android        # 웹빌드 + 모바일용 모델(300MB) 제외 + cap sync
cd android && .\gradlew.bat bundleRelease   # 서명된 AAB
```
- 결과: `android/app/build/outputs/bundle/release/app-release.aab`
- 서명: `android/yei-upload.keystore`(별칭 `yei`), 비번은 `android/keystore.properties`.
- 🔐 **키스토어·비번 백업 필수**(USB/비번관리자). 단 Play App Signing 수락 시 업로드키 분실해도 재설정 가능.

### B. 플레이 콘솔 — 앱 + 내부 테스트
1. Play Console → 앱 만들기(이름 Yei, 무료).
2. 패키지명 `com.yei.idphoto` (첫 AAB 업로드로 고정, 변경 불가).
3. 테스트 → 내부 테스트 → 새 버전 → AAB 업로드.
4. 테스터 탭에 본인 구글 계정 추가 → **참여 링크(opt-in URL)** 확보.

### C. 인앱 상품 (AAB 1회 업로드 후 가능)
1. 수익 창출 → 제품 → 인앱 상품 → 만들기.
2. 상품 ID **`yei_lifetime`** (← `src/iap.ts` `PRODUCT_ID` 와 동일해야 함).
3. 유형 **일회성(non-consumable)**, 가격 **₩4,900**, **활성화**.

### D. 라이선스 테스터 (무청구 테스트)
- Play Console 설정 → 라이선스 테스트 → 본인 구글 계정 추가 → 테스트 구매 무청구.

### E. RevenueCat
1. RevenueCat 프로젝트 생성 → Play Store 앱 추가(`com.yei.idphoto`).
2. 구글 서비스 계정 JSON 연결(RevenueCat 안내대로 권한 부여).
3. Products `yei_lifetime` → Entitlements `premium` 연결 → Offerings(current)에 패키지 추가.
4. API keys → Google 의 `goog_...` 공개키 복사.

### F. 키 넣고 재빌드
1. `src/iap.ts` `RC_API_KEY` 에 `goog_...` 입력. *(베타 무변경 원칙 — 이 한 줄 변경은 본인 확인 후)*
2. `npm run sync:android` → `gradlew bundleRelease` → 내부 테스트에 **새 버전 업로드**.

### G. 폰 테스트
- 내부 테스트 참여 링크 옵트인 → **플레이에서 설치**(USB 설치본 X).
- 무료 5회 → 6회째 페이월 → "광고 없이 평생 이용" → (테스트계정 무청구) → 즉시 무제한.
- 삭제·재설치 → "구매 복원" → 무제한.

## 2. 성공 판정
