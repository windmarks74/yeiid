# Play 콘솔 데이터 보안(Data Safety) 양식 — Yei 작성 가이드

> 위치: Play Console → **앱 콘텐츠(App content) → 데이터 보안(Data safety)**
> 원칙: 사진은 **수집 안 함**(온디바이스). 결제/식별자는 Google Play 결제 + RevenueCat,
> 진단 데이터는 Google ML Kit SDK(얼굴인식·인물분리)를 통해 처리.
> ⚠️ 정확성: 아래는 현재 구현 기준 권장 답안. 변경 시 갱신하고, RevenueCat 공식
> 데이터 보안 안내(https://www.revenuecat.com/docs)와 ML Kit 데이터 공개 문서
> (https://developers.google.com/ml-kit/android-data-disclosure)도 한 번 교차 확인하세요.

## 1단계 — 개요 질문
| 질문 | 답 |
|---|---|
| 앱이 필수 사용자 데이터 유형을 **수집 또는 공유**하나요? | **예** (결제/식별자 때문) |
| 수집하는 모든 사용자 데이터가 **전송 중 암호화**되나요? | **예** (HTTPS) |
| 사용자가 **데이터 삭제를 요청**할 방법을 제공하나요? | **예** → 이메일 `shhwang0424@gmail.com` (앱 데이터는 앱 삭제 시 함께 삭제됨도 명시) |

## 2단계 — 수집하는 데이터 유형 (체크할 것만)

### ✅ 구매 기록 (App activity → Purchase history)
- 수집됨: **예**
- 공유됨: **아니요** *(RevenueCat·Google은 우리 대신 처리하는 서비스 제공업체 → "공유" 아님)*
- 목적: **앱 기능**(구매·복원). ※ 광고/마케팅 **아님**
- 사용자 ID에 연결: **아니요** (로그인 없음, 익명 ID)
- 수집 필수 여부: **필수**

### ✅ 기기 또는 기타 ID (Device or other IDs)
- 수집됨: **예** *(RevenueCat 익명 App User ID / IP 등 구매 귀속·복원용 + ML Kit SDK의 앱·기기 식별자)*
- 공유됨: **아니요** (서비스 제공업체)
- 목적: **앱 기능** (+ 필요시 사기 방지·보안)
- 사용자 ID에 연결: **아니요**

### ✅ 앱 정보 및 성능 → 진단 (App info and performance → Diagnostics)
- 수집됨: **예** *(Google ML Kit 온디바이스 얼굴인식·인물분리 SDK가 기기 메타데이터·성능 진단 데이터를 Google로 전송 — 사진 자체는 전송 안 됨)*
- 공유됨: **아니요** (서비스 제공업체)
- 목적: **앱 기능, 분석** (SDK 품질·성능 개선)
- 사용자 ID에 연결: **아니요**
- 수집 필수 여부: **필수** (SDK 내장 동작, 옵트아웃 불가)
- 근거: https://developers.google.com/ml-kit/android-data-disclosure — `face-detection`, `segmentation-selfie` 항목 기준. 온디바이스 API라도 진단·식별자 신고 필요.

## 3단계 — 수집 안 함 (전부 "아니요" / 체크 해제)
- **사진 및 동영상** → ❌ 수집 안 함 *(앱의 핵심 강점 — 기기 내에서만 처리, 외부 전송 없음)*
- **개인정보**(이름·이메일·전화 등) → ❌ (회원가입·로그인 없음)
- **위치**(대략/정밀) → ❌ (GPS 미사용)
- **금융정보**(카드번호 등) → ❌ (결제는 Google이 처리, 앱은 카드정보 안 받음)
- 연락처·메시지·캘린더·건강·웹 검색기록 등 → ❌ 전부 아님

## 4단계 — 보안 관행(Security practices)
| 항목 | 답 |
|---|---|
| 전송 중 데이터 암호화 | **예** |
| 사용자가 데이터 삭제 요청 가능 | **예** → `shhwang0424@gmail.com` |
| 가족 정책(아동 대상) 준수 약속 | 해당 없음(아동 대상 앱 아님) |
| 독립 보안 검토 | 선택(공란 가능) |

## 메모
- "공유(shared)"는 *제3자가 자기 목적으로 쓰도록 넘기는 것*을 뜻함. RevenueCat/Google은
  우리 대신 결제를 처리하는 **수탁사**라 일반적으로 "공유 아니요"로 답함.
- Google Play 결제 자체가 수집하는 데이터는 구글이 신고 → 우리는 **RevenueCat SDK가
  처리하는 것**(구매기록·식별자)만 신고하면 됨.
- **ML Kit**: 온디바이스 API지만 Google 공식 문서가 기기 메타데이터·앱 식별자·성능 진단
  데이터의 Data Safety 신고를 요구함 → 위 "진단" 항목 필수. 사진은 여전히 전송 안 됨.
- **RevenueCat 호출 시점**: 구매 시점만이 아니라 **앱 실행 시 구매 상태 확인**(entitlement
  조회)으로도 식별자·IP가 전송됨 — "수집됨: 예"가 전 사용자 대상임을 전제로 답할 것.
- 사진을 "수집 안 함"으로 정확히 신고하는 게 핵심 — 온디바이스 처리라 사실 그대로.
